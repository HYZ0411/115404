# 會議記錄：畢業專題進度與 GitHub 工具教學

**日期：** 2026年1月19日
**主持人：** 張浩洋
**出席人員：** 吳昶毅、姚永浚、劉凱閎、張浩洋（全員準時出席）

---

### 一、 會議重點摘要
1. **GitHub 工具導入：** 為了方便評審查看與版本管理，團隊將正式啟用 GitHub 上傳專案資料。
2. **資料格式轉換：** 由於 Notion 直接上傳 GitHub 可能會跑版，規定市場調查資料須先轉成 Word 格式再進行排版上傳。
3. **專案管理流程：** 本週起將嚴格執行進度更新，確保期中與期末考期間能減輕專案負擔。

---

### 二、 個人任務分配
* **張浩洋：** 製作市場調查 Word 範本，並指導成員使用 GitHub。專題文件製作。
* **全體成員：** 於週三至週六將負責的市場調查內容貼至 Word。
* **吳昶毅：** 規劃健身計畫草案，並於週二前討論。
* **姚永浚：** 進行設計稿改版與模板套用。
* **劉凱閎：** 完成 50 個健身動作示範製作（進度為每日 10 個）。

---

### 三、 技術教學要點
* **開發環境設定：** 成員需下載 VS Code 與 Git 環境。
* **GitHub 操作：**
    - 使用學生帳號登入 GitHub。
    - 使用 `git clone` 指令將專案複製至本地電腦。
    - 解決登入驗證問題（建議使用 Token 或相關驗證工具）。

---

### 四、 後續行動項目
* **週末小組會議：** 週六或週日下午將再次集合，確保所有成員都能成功將資料推送（Push）至 GitHub。
* **進度追蹤：** 吳昶毅需於週二前確認健身計畫內容。
