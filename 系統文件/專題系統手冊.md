# 專題系統手冊

專題名稱：「Alchemy Workout 健金術」

## 第一章 前言

### 1-1 背景介紹
在傳統健身的漫長征途中，多數參與者並非敗於體力的枯竭，而是溺於「回饋沙漠」中的心理消磨。

「鍊金術」的本質是物質的轉化與昇華。在 Alchemy Workout 健金術 的設計語境中，使用者的每一滴汗水（Raw Material），都不應只是無聲流失的代謝產物，而應被視為淬煉成數據資產與心理成就（Gold）的原始原料。

目前的數位健康市場正面臨嚴峻的「高流失率」挑戰。數據顯示，約有 80% 的新進會員在健身啟程後的五個月內選擇放棄。這背後的行為心理學關鍵在於「生理反饋延遲」：肌肉組織的肥大或體脂率的下降，往往需要數週甚至數月的生理週期方能顯現；然而，人類大腦的獎勵系統（Dopamine System）卻極度依賴「即時回饋」。當投入的成本與感知的收穫出現巨大的時間鴻溝，用戶便會因信心磨損而產生行為斷裂。

本系統導入「遊戲化八角框架 (Octalysis)」，旨在縮短這段回饋路徑，將長期、隱性的生理變化，轉化為短期、顯性的心理酬賞，藉此抵銷生理延遲帶來的負面影響。

### 1-2 動機
我們觀察到一個有趣的心理對比：在遊戲場景中，玩家為了能看見角色「數值成長」往往展現出驚人的毅力與耐心；但在現實健身中，這種動力卻難以長期維繫。

原因在於「進步難以被即時察覺」。在現實世界裡，訓練所累積的成果往往隱藏在感官之外。當用戶投入了大量的時間與體力，卻無法即時從鏡子裡看到明顯的身形變化時，便會產生強烈的自我懷疑。這種「我練了這麼久，到底有沒有變得更好？」的迷惘，是導致大多數人在跨越習慣養成門檻前，就因為心理磨損而選擇放棄的主因。

因此，「健金術」的開發動機，其核心機制在於建立一套「現實生活中的能力值面板」。本系統全面導入「遊戲化八角框架」的八項核心動力，將複雜的行為心理學轉化為具體，讓「進步」從隱形變為透明。

### 1-3 系統目的與目標
本系統旨在透過「遊戲化八角框架」的核心動力，解決不同階段使用者在健身旅程中的障礙。我們將針對三大目標客群，設定對應的系統開發目標：
* 針對入門新手：系統將使用者每次的健身紀錄都轉化爲「各主要肌群成長雷達圖」，目標是透過視覺化的身體素質模型，讓新手能立即察覺那些肉眼尚不可見的微小進步。
* 針對自我挑戰者：建構精簡的「訓練資產管理」，提供「訓練目標傾向圓餅圖」與「訓練成長橫條圖」。目標是讓挑戰者能快速掌握自己的訓練配比與長期成長趨勢。
* 針對社群互動：建立「健身夥伴社群」。目標是確保新手在面臨訓練困惑或動力低迷時，能隨時獲得真人指導與支持。透過夥伴間的經驗交流，新手能有效克服健身門檻，減少孤軍奮戰的挫折感。

本專案旨在透過行為心理學與數據工程的交織，重塑健身體驗。以下為本系統與傳統競品之維度對比：

| 比較維度 | 傳統健身 App (Traditional Apps) | 健金術 (Alchemy Workout) |
| :--- | :--- | :--- |
| **動機觸發** | **外部推力導向**：依賴推播通知與用戶自律，容易因意志力耗損而中斷。 | **內在驅動導向**：導入「八角框架」，利用使命感、擁有感等核心動力誘發自主行為。 |
| **數據呈現** | **線性數據紀錄**：以清單或簡單折線圖顯示體重、次數，缺乏情感連結。 | **多維能力面板**：將汗水轉化為「數據黃金」，以雷達圖與能力值視覺化展現成長感。 |
| **挫折處理** | **負向壓力機制**：一旦中斷紀錄便產生斷層，讓用戶因愧疚感而產生放棄念頭。 | **資產留存機制**：強調已累積的「訓練資產」，並透過 AI 調節難度，降低心理摩擦力。 |
| **社群互動** | **競爭性排行榜**：單純的排名競爭，容易讓新手感到挫折或產生社交焦慮。 | **互助師徒生態**：建立雙向回饋機制，老手獲得聲望、新手獲得引導，形成共榮圈。 |
| **決策輔助** | **靜態計畫書**：內容固定，若遇到器材短缺或身體不適，計畫便難以執行。 | **AI 動態教練**：根據即時生理反饋與環境限制（如器材缺失）即時彈性調整訓練戰術。 |

### 1-4 預期效果
本系統透過遊戲化邏輯與數據建模，預期在產品營運與用戶行為上達成以下成果：
* 健金術行動健身平台：建構一套功能完備的移動端應用程式，透過精簡的數據儀表板即時呈現使用者的訓練總量、肌群強度與成長趨勢。
* 動態能力視覺化系統：提供支援多維度分析（雷達圖、圓餅圖、橫條圖）的「健身分析報告」，將顯性的數據資產轉化為持續運動的心理動力。
* 雙向互助的師徒社群生態： 藉由「健身夥伴社群」，為新手營造一個不再孤單的成長環境。當初學者遇到難題時，能獲得熱心夥伴的即時應援與解惑。這不僅顯著降低了新手的焦慮感，更透過「互助獲得聲望」的良性循環，建立起一個具備高度向心力與知識分享氛圍的健身圈。
* AI 智能教練模組：自動分析訓練紀錄以主動微調訓練強度，並在缺器材時即時提供替代建議。藉此消除運動時的決策摩擦，確保用戶能流暢的執行訓練計畫。

## 第二章 營運計畫

### 2-1 可行性分析
1. 技術可行性：
本系統之技術可行性並非建立在傳統的「代碼堆砌（Code-heavy implementation）」之上，而是基於高階軟體工程的決策力與架構設計能力。在 AI 工具高度成熟的當下，本團隊將開發重心從基礎的「程式語法編寫」轉向更具商業價值的「問題定義與領域建模」。透過掌握更深層的軟體工程技巧，我們證明了即便在資源有限的專題開發環境下，亦能產出具備商用級複雜度且邏輯嚴密的系統。

在開發流程的頂層設計中，我們導入了 DDD（領域驅動設計）的核心方法論，並透過「事件風暴 (Event Storming)」模型，將「健金術」中複雜的業務場景——如用戶註冊流程、健身紀錄動態權重計算、以及遊戲化獎勵觸發等，進行精確的程式分析。這使我們能識別出關鍵的邊界事件與限界上下文（Bounded Context），確保系統的各項功能模組具備高度的內聚性與擴展性。

隨後，這些抽象的領域邏輯會進一步轉化為 BDD（行為驅動開發）的技術指令。透過 Gherkin 規格書，我們將使用者行為與系統行為進行精準對齊。在此模式下，開發者的角色已從傳統的「編碼者（Coder）」升級為「系統架構策略師（System Strategist）」。具體的實作任務則交由具備代理型能力的 Claude Code 進行閉環執行，並由 Gemini API 根據我們設定的「專業訓練動作庫」提供智能導航與決策輔助。

這種「決策權力核心化、實作自動化」的開發路徑，不僅消除了傳統開發中常見的溝通落差與語法錯誤，更大幅降低了因人為介入而產生的邏輯摩擦感。透過 Flutter 跨平台框架與 Firebase 雲端架構的支撐，我們將技術可行性從「功能實現」提升到了「架構美學」與「工程效率」的高度結合，充分體現了資管系學生在 AI 時代駕馭複雜技術棧、主導系統演進的卓越能力。

### 2-2 商業模式 (Business Model Canvas)

本專案採用精實創業畫布 (Lean Canvas) 邏輯，構建高價值、低維運成本的商業閉環。

#### 1. 價值主張 (Value Propositions)
*   **可視化的成就感系統**：將看不見的生理變化，轉化為看得見的數據圖表（雷達圖、成長曲線）。
*   **克服回饋延遲的心理引擎**：利用遊戲化機制，填補健身初期「付出多、回報少」的心理落差，解決高流失率痛點。

#### 2. 目標客群 (Customer Segments)
*   **主要客群**：16-35 歲的大學生與初階白領。
*   **特徵**：有健身意願但缺乏恆心、需要外部激勵機制、曾經放棄過健身計畫的「迷惘新手」。

#### 3. 通路 (Channels)
*   **首發曝光**：全國大專院校資訊應用服務創新競賽 (InnoServe) 舞台。
*   **精準獲客**：透過「體育老師人脈網」將 App 導入大專院校體育課程。
*   **地面推廣**：校園健身房地推活動。

#### 4. 顧客關係 (Customer Relationships)
*   **AI 智能教練**：提供 24/7 的即時反饋與陪伴感。
*   **社群互助**：建立「師徒制」社群，透過同儕壓力與鼓勵維持黏著度。

#### 5. 收益流 (Revenue Streams)
*   **訂閱制**：**$99 TWD/月** (進階數據分析功能)。
*   **廣告收入**：非侵入式的激勵廣告（觀看廣告以解鎖特定圖表皮膚）。

#### 6. 關鍵資源 (Key Resources)
*   **核心技術**：AI x BDD 開發技術能力。
*   **智財資產**：八角框架 (Octalysis) 轉化之遊戲化邏輯演算法。
*   **硬體資源**：校園免費運算資源與 VM。

#### 7. 關鍵活動 (Key Activities)
*   **產品迭代**：基於數據的軟體功能優化。
*   **自動化營運**：利用 n8n 維護內容行銷矩陣。
*   **社群經營**：舉辦線上挑戰賽以活化社群。

#### 8. 關鍵合作夥伴 (Key Partners)
*   **營養補充品廠商**：進行互惠導購與分潤合作。
*   **學校體育室**：作為場域驗證合作夥伴。

#### 9. 成本結構 (Cost Structure)
*   **極簡運維**：導入 **n8n 自動化工具**，自動爬取健身新知並生成社群貼文，大幅降低行銷人力成本。
*   **浮動成本**：LLM (大型語言模型) API Token 費用（隨用量調整）。
*   **固定成本**：極低的網域與伺服器租賃費（初期免費）。

### 2-3 市場分析與 STP 策略

#### 市場區隔與目標 (Segmentation & Targeting)
根據我們執行的 **30 人定性前導研究** 顯示，新手放棄健身的主因並非「不知道怎麼練」，而是「感覺不到進步」(回饋感低落)。因此，我們避開與專業教練工具的競爭，專注鎖定**「追求成就感與樂趣」的健身入門者**。

#### 定位 (Positioning)
*   **競品定位**：市面上的 Keep 或 Nike Training Club 定位為**「最專業的教練」**，強調課程與權威。
*   **我方定位**：本產品定位為**「最好玩的健身夥伴」**，強調**「數據可視化」**與**「遊戲化激勵」**。我們不教你深奧的解剖學，我們讓你對每一次的深蹲感到上癮。

### 2-4 競爭力分析 (SWOT-TOWS)

我們透過 TOWS 矩陣分析，確立「以技術換取空間」的競爭策略。

#### 第一部分：SWOT 清單
*   **優勢 (Strengths)**：AI x BDD 帶來的高效產能、八角框架心理機制、極低的營運成本結構。
*   **劣勢 (Weaknesses)**：缺乏行銷預算、團隊人數少 (4人)、初期無品牌知名度。
*   **機會 (Opportunities)**：AI 工具普及降低創業門檻、Z 世代對遊戲化與數據紀錄的高接受度。
*   **威脅 (Threats)**：大型競品 (Keep/Nike) 的資源夾殺、日益嚴格的資安隱私法規。

### 2-5 定價策略：手搖飲定價法 (The Bubble Tea Pricing)

我們制定了極具破壞力的定價策略：**訂閱費 $99 TWD/月**。

*   **心理錨點**：此價格等同於大學生日常消費的一杯高單價手搖飲。這創造了一個強力的行銷話術：「少喝一杯飲料，換取一個月的身材管理教練」。
*   **無痛轉化**：相較於競品動輒 $300-$500 的月費，低於 $100 的定價大幅降低了學生的決策門檻 (Decision Friction)。
*   **成本覆蓋**：經精算，即便在 $99 元的低價下，只要用戶每日使用頻率在正常範圍內，營收仍足以覆蓋 LLM API 的 Token 浮動成本與伺服器開銷，確保商業模式的永續性。
