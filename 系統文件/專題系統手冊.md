# 專題系統手冊

專題名稱：「Alchemy Workout 健金術」

## 第一章 前言

### 1-1 背景介紹
在傳統健身的漫長征途中，多數參與者並非敗於體力的枯竭，而是溺於「回饋沙漠」中的心理消磨。

「鍊金術」的本質是物質的轉化與昇華。在 Alchemy Workout 健金術 的設計語境中，使用者的每一滴汗水（Raw Material），都不應只是無聲流失的代謝產物，而應被視為淬煉成數據資產與心理成就（Gold）的原始原料。

目前的數位健康市場正面臨嚴峻的「高流失率」挑戰。數據顯示，約有 80% 的新進會員在健身啟程後的五個月內選擇放棄。這背後的行為心理學關鍵在於「生理反饋延遲」：肌肉組織的肥大或體脂率的下降，往往需要數週甚至數月的生理週期方能顯現；然而，人類大腦的獎勵系統（Dopamine System）卻極度依賴「即時回饋」。當投入的成本與感知的收穫出現巨大的時間鴻溝，用戶便會因信心磨損而產生行為斷裂。

本系統導入「遊戲化八角框架 (Octalysis)」，旨在縮短這段回饋路徑，將長期、隱性的生理變化，轉化為短期、顯性的心理酬賞，藉此抵銷生理延遲帶來的負面影響。

### 1-2 動機
我們觀察到一個有趣的心理對比：在遊戲場景中，玩家為了能看見角色「數值成長」往往展現出驚人的毅力與耐心；但在現實健身中，這種動力卻難以長期維繫。

原因在於「進步難以被即時察覺」。在現實世界裡，訓練所累積的成果往往隱藏在感官之外。當用戶投入了大量的時間與體力，卻無法即時從鏡子裡看到明顯的身形變化時，便會產生強烈的自我懷疑。這種「我練了這麼久，到底有沒有變得更好？」的迷惘，是導致大多數人在跨越習慣養成門檻前，就因為心理磨損而選擇放棄的主因。

因此，「健金術」的開發動機，其核心機制在於建立一套「現實生活中的能力值面板」。本系統全面導入「遊戲化八角框架」的八項核心動力，將複雜的行為心理學轉化為具體，讓「進步」從隱形變為透明。

### 1-3 系統目的與目標
本系統旨在透過「遊戲化八角框架」的核心動力，解決不同階段使用者在健身旅程中的障礙。我們將針對三大目標客群，設定對應的系統開發目標：
* 針對入門新手：系統將使用者每次的健身紀錄都轉化爲「各主要肌群成長雷達圖」，目標是透過視覺化的身體素質模型，讓新手能立即察覺那些肉眼尚不可見的微小進步。
* 針對自我挑戰者：建構精簡的「訓練資產管理」，提供「訓練目標傾向圓餅圖」與「訓練成長橫條圖」。目標是讓挑戰者能快速掌握自己的訓練配比與長期成長趨勢。
* 針對社群互動：建立「健身夥伴社群」。目標是確保新手在面臨訓練困惑或動力低迷時，能隨時獲得真人指導與支持。透過夥伴間的經驗交流，新手能有效克服健身門檻，減少孤軍奮戰的挫折感。

本專案旨在透過行為心理學與數據工程的交織，重塑健身體驗。以下為本系統與傳統競品之維度對比：

| 比較維度 | 傳統健身 App (Traditional Apps) | 健金術 (Alchemy Workout) |
| :--- | :--- | :--- |
| **動機觸發** | **外部推力導向**：依賴推播通知與用戶自律，容易因意志力耗損而中斷。 | **內在驅動導向**：導入「八角框架」，利用使命感、擁有感等核心動力誘發自主行為。 |
| **數據呈現** | **線性數據紀錄**：以清單或簡單折線圖顯示體重、次數，缺乏情感連結。 | **多維能力面板**：將汗水轉化為「數據黃金」，以雷達圖與能力值視覺化展現成長感。 |
| **挫折處理** | **負向壓力機制**：一旦中斷紀錄便產生斷層，讓用戶因愧疚感而產生放棄念頭。 | **資產留存機制**：強調已累積的「訓練資產」，並透過 AI 調節難度，降低心理摩擦力。 |
| **社群互動** | **競爭性排行榜**：單純的排名競爭，容易讓新手感到挫折或產生社交焦慮。 | **互助師徒生態**：建立雙向回饋機制，老手獲得聲望、新手獲得引導，形成共榮圈。 |
| **決策輔助** | **靜態計畫書**：內容固定，若遇到器材短缺或身體不適，計畫便難以執行。 | **AI 動態教練**：根據即時生理反饋與環境限制（如器材缺失）即時彈性調整訓練戰術。 |

### 1-4 預期效果
本系統透過遊戲化邏輯與數據建模，預期在產品營運與用戶行為上達成以下成果：
* 健金術行動健身平台：建構一套功能完備的移動端應用程式，透過精簡的數據儀表板即時呈現使用者的訓練總量、肌群強度與成長趨勢。
* 動態能力視覺化系統：提供支援多維度分析（雷達圖、圓餅圖、橫條圖）的「健身分析報告」，將顯性的數據資產轉化為持續運動的心理動力。
* 雙向互助的師徒社群生態： 藉由「健身夥伴社群」，為新手營造一個不再孤單的成長環境。當初學者遇到難題時，能獲得熱心夥伴的即時應援與解惑。這不僅顯著降低了新手的焦慮感，更透過「互助獲得聲望」的良性循環，建立起一個具備高度向心力與知識分享氛圍的健身圈。
* AI 智能教練模組：自動分析訓練紀錄以主動微調訓練強度，並在缺器材時即時提供替代建議。藉此消除運動時的決策摩擦，確保用戶能流暢的執行訓練計畫。

## 第二章 營運計畫

### 2-1 可行性分析

1. 技術可行性：
本系統之技術可行性並非建立在傳統的「代碼堆砌（Code-heavy implementation）」之上，而是基於高階軟體工程的決策力與架構設計能力。在 AI 工具高度成熟的當下，本團隊將開發重心從基礎的「程式語法編寫」轉向更具商業價值的「問題定義與領域建模」。透過掌握更深層的軟體工程技巧，我們證明了即便在資源有限的專題開發環境下，亦能產出具備商用級複雜度且邏輯嚴密的系統。在開發流程的頂層設計中，我們導入了 DDD（領域驅動設計）的核心方法論，並透過「事件風暴 (Event Storming)」模型，將「健金術」中複雜的業務場景——如用戶註冊流程、健身紀錄動態權重計算、以及遊戲化獎勵觸發等，進行精確的程式分析。這使我們能識別出關鍵的邊界事件與限界上下文（Bounded Context），確保系統的各項功能模組具備高度的內聚性與擴展性。

隨後，這些抽象的領域邏輯會進一步轉化為 BDD（行為驅動開發）的技術指令。透過 Gherkin 規格書，我們將使用者行為與系統行為進行精準對齊。在此模式下，開發者的角色已從傳統的「編碼者（Coder）」升級為「系統架構策略師（System Strategist）」。具體的實作任務則交由具備代理型能力的 Claude Code 進行閉環執行，並由 Gemini API 根據我們設定的「專業訓練動作庫」提供智能導航與決策輔助。這種「決策權力核心化、實作自動化」的開發路徑，不僅消除了傳統開發中常見的溝通落差與語法錯誤，更大幅降低了因人為介入而產生的邏輯摩擦感。透過 Flutter 跨平台框架與 Firebase 雲端架構的支撐，我們將技術可行性從「功能實現」提升到了「架構美學」與「工程效率」的高度結合，充分體現了資管系學生在 AI 時代駕馭複雜技術棧、主導系統演進的卓越能力。

2. 市場可行性：
「健金術」的市場可行性建立在數位健康產業從「量化數據」轉向「情緒價值」的技術拐點。根據中華民國教育部體育署《113年運動現況調查》，台灣規律運動人口比例已攀升至 35.3%。顯示健康意識已從少數愛好轉向全民剛需，然而運動習慣能否持續的關鍵在於如何跨越「回饋延遲」的心理障礙。為此，美國運動醫學會（ACSM）發布的《2026 全球健身趨勢報告》明確將「行動運動應用程式」列為全球第四大核心趨勢，指出市場急需具備便利性、結構化與自我監控能力的數位工具。

這項需求在 Glance 趨勢報告《Fitness App Trends 2025: What Users Really Want From Their Workout Apps》中得到了進一步驗證。報告指出，現代使用者極度渴望結合「遊戲化機制」（如連續天數、等級進階）與「無縫數據追蹤」的功能，以打破單調的訓練過程並提高長期留存率。相對於功能單一的傳統紀錄 App，「健金術」運用「八角框架」將汗水資產化的機制，不僅解決了新手期的回饋沙漠問題，更透過師徒制社群創造了強大的關係黏著度。在 2026 年競爭激烈的數位健康藍海中，本系統憑藉其獨特的行為心理學優勢與數據建模技術，展現出極強的市場替代力與持續獲利潛力。

### 2-2 商業模式 (Business Model Canvas)

#### 1. 價值主張 (Value Propositions)
*   即時心理酬賞：透過「八角框架」遊戲化機制，將數週才能顯現的生理變化，轉化為「肉眼可見」的即時數據成長。
*   數據資產化：將每一次訓練定義為「數據黃金」，透過資產留存機制（能力面板、雷達圖）建立用戶的成就感護城河。
*   消除決策摩擦：AI 智能教練能根據即時反饋與環境限制（如器材缺失）動態調整計畫，確保用戶無痛執行訓練。
*   互助共榮圈：獨創「師徒生態」，讓老手獲得聲望、新手獲得引導，將孤軍奮戰轉化為具備歸屬感的社群修煉。

#### 2. 目標客群 (Customer Segments)
*   迷惘的健身新手：處於「回饋沙漠」期，因短期看不見成效而容易產生挫折感、動搖意志的初學者。
*   數據導向的自我挑戰者：重視邏輯分析，需要精準的「訓練資產管理」與長期成長趨勢追蹤的進階訓練者。
*   尋求連結的社群運動者：依賴社交互動驅動，偏好在具備回饋與互助機制的環境中維持運動習慣的用戶。

#### 3. 通路 (Channels)
*   行動應用程式商店：App Store 與 Google Play 作為主要的下載與獲客管道。
*   視覺化社交傳播：透過用戶分享具備美感的「能力值雷達圖」至 Instagram、Threads 等社群平台，達成病毒式擴散。
*   線下健身房合作：與工作室合作，將系統作為其學員課後管理的數位延伸工具。

#### 4. 顧客關係 (Customer Relationships)
*   自動化個人助理：AI 教練提供 24/7 的即時指導與難度調節，建立高度個人化的依賴感。
*   共同創造與社群：透過師徒制與夥伴社群，讓用戶從單純的系統使用者轉化為內容與支援的提供者。
*   長期鎖定效應：藉由「資產留存」邏輯，用戶在系統內累積的「數據黃金」越高，轉移至其他競品的成本就越大。

#### 5. 收益流 (Revenue Streams)
*   Freemium 訂閱制：基礎紀錄與視覺化免費，進階的「AI 動態教練模組」與「深度資產分析報告」採月/年費訂閱。
*   數位成就資產： 提供特殊的「遊戲化外觀」或「聲望勳章」供用戶購買或達成任務獲得。
*   B2B 數據服務： 提供去識別化的健體數據與行為模式分析給健身器材商或保險業。

#### 6. 關鍵資源 (Key Resources)
*   專業體育領域人才：由具備體育專業背景之組員，針對「新手與高手」、「訓練時間長短」、「訓練目標」及「現有器材」等多維度指標，研發產出 50 套以上之深度訓練計畫。
*   高階動作資料庫：擁有超過 500 項標準化健身動作之龐大數據資產，作為系統精準決策、AI 導航與視覺化分析之底層邏輯。
*   技術與雲端架構：整合 Flutter 跨平台開發與 Firebase 雲端資料庫，確保高頻率訓練紀錄之數據穩定性與擴展性。

#### 7. 關鍵活動 (Key Activities)
*   內在動機觸發機制：以「遊戲化八角框架 (Octalysis)」為核心研究方向，將枯燥的物理重複訓練轉化為增強使用者核心動能（如使命感、成就感）的驅動行為。
*   系統架構演進與維護：持續優化 DDD 領域建模與 BDD 自動化實作，確保「汗水轉化為資產」的邏輯運算精準且流暢。
*   社群與師徒生態運營：管理並引導高聲望用戶與新手之間的互助循環，藉此建立具備高向心力的健身修煉圈。

#### 8. 關鍵合作夥伴 (Key Partners)
*   技術供應商： Google (Firebase/Gemini)、Anthropic (Claude) 提供底層技術支持。
*   健身領域專家： 專業健身教練與運動心理學家，確保訓練建議的專業性。
*   健身場館與工作室： 作為推廣的前線節點，協助獲取核心種子用戶。

#### 9. 成本結構 (Cost Structure)
*   研發與技術成本： 雲端伺服器（Firebase）託管費、AI API 調用費、軟體工程研發人力。
*   行銷與獲客成本： 社群廣告投放、KOL/健身部落客合作費用。
*   運營與內容成本： 專業教練顧問費、社群維護人力。

### 2-3 市場分析與 STP 策略

#### Segmentation 市場區隔：
* 心理動機維度：
    * 短效回饋型：極度依賴即時正向回饋（如遊戲等級、視覺獎勵），若無即時成效容易在 1-3 個月內放棄。
    * 長效目標型：具備強大意志力，能忍受生理延遲，追求長期的體能指標進步。
* 數據應用維度：
    * 輕量紀錄者：僅需基本數據（如體重、次數），不喜歡複雜的輸入流程。
    * 深度分析者：視數據為資產，追求多維度（肌群比例、強度變化）的專業報告。
* 社交參與維度：
    * 孤軍奮戰型：偏好自主訓練，不喜歡社交干擾，僅需 AI 與系統輔助。
    * 群體驅動型：需透過與他人互動、獲得聲望或幫助他人來維持運動動力。
* 決策習慣維度：
    * 依賴導航型：面對器材短缺或身體疲勞時容易不知所措，需要 AI 即時調整計畫。
    * 自主導向型：習慣自行規劃訓練內容，僅將 App 作為輔助紀錄工具。

#### Targeting 目標市場：
* 受困於「回饋沙漠」的迷惘新手：這類用戶是市場中流失率最高（約 80%）的族群。他們需要將「隱性生理變化」轉化為「顯性數據資產」，是「遊戲化八角框架」最能發揮價值的對象。
* 追求極致效率的「數據資產管理員」：針對具備一定健身基礎，但對傳統 App 線性數據感到無趣的進階者。他們重視「訓練資產化」與「AI 動態調節」，藉此消除訓練中的決策摩擦。
* 尋求「成就感認同」的社群推動者：瞄準喜愛透過分享「能力雷達圖」展現自我、並願意在師徒生態中累積聲望的資深用戶，他們將成為平台內容與社群黏著度的核心。

#### Positioning 市場定位：
*   便利性：透過 AI 智能教練消除決策摩擦。當面對器材短缺或體力波動時，系統能自動提供替代方案與即時調整強度，讓使用者無需具備專業課表規劃知識，即可流暢且高效地執行訓練。
*   社交性：建立雙向互助的師徒制生態。打破傳統 App 單純的數據競爭，透過新手獲得引導、老手累積聲望的機制，將枯燥的重複訓練轉化為具備歸屬感與心理支持的社群連結。
*   低門檻：運用多維度能力值面板將進步視覺化。透過雷達圖與數據金幣機制，將難以察覺的生理變化轉化為直覺的心理酬賞，讓新手即便不具備健身專業知識，也能一眼看見成長並跨越回饋沙漠。

## 第三章 系統架構

### 3-1 系統架構
分析出多數健身平台數據呈現過於線性且缺乏心理誘因，本系統旨在透過遊戲化八角框架與師徒制社群，將枯燥的訓練轉化為具備情感價值的數位資產。本系統採用 Serverless 無伺服器架構，整合 Flutter、Firebase 與 Vertex AI 技術棧，確保系統在具備快速落地上架能力的同時，亦達成商用級的數據安全性與運算效率。系統架構主要分為以下三個部分：

1. 前端介面與行動端整合：本系統採用 Flutter 跨平台框架開發，並透過 FlutterFire 插件深度串接 Firebase 官方服務。前端除負責呈現雷達圖、圓餅圖與折線圖等視覺化指標外，亦整合了 Firebase Cloud Messaging（FCM）推播機制。這確保了師徒社群間的訓練提醒與成就回饋能即時傳達。此外，系統導入 App Check 機制，從設備端限制只有合法的 App 實例能呼叫後端資源，有效防止 API 遭惡意盜刷。

2. 雲端後端服務與安全數據管理：採用 Firebase 作為核心雲端基礎設施。透過 Firebase Authentication 實作身分驗證，並利用 Cloud Firestore 達成訓練數據與社交資產的即時同步。為了保護機密金鑰（如 API Keys），系統採用 Cloud Functions 搭配 Secret Manager 作為後端邏輯中樞，將所有敏感運算與外部請求隔絕於雲端環境中。此設計確保了即便 App 遭反編譯，核心金鑰仍能維持絕對安全，符合 App Store 的資安規範。

3. 智慧導航與 AI 轉化引擎：本系統整合 Vertex AI in Firebase 實作正式環境的生成式 AI 功能。在開發階段透過 Google AI Studio 進行 Prompt 與回應品質的精準測試，確保 AI 建議的專業度。在執行階段，系統能根據 Firestore 中的即時紀錄，由 Cloud Functions 觸發 AI 進行雙階段輔助：訓練中的動態計畫調整以及訓練後的深度成長報告。此架構將隱性的汗水數據轉化為顯性的數位資產，透過 AI 的精準引導大幅提升使用者的心理酬賞與長期留存率。

### 3-3 使用標準與工具
