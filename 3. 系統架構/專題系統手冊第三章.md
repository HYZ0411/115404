# 專題系統手冊

## 第3章 系統架構

### 3-1 系統架構

分析出多數健身平台數據呈現過於線性且缺乏心理誘因，本系統旨在透過遊戲化八角框架與師徒制社群，將枯燥的訓練轉化為具備情感價值的數位資產。系統採用 Serverless 無伺服器架構，整合 Flutter、Firebase 與 Google AI 技術棧。為確保開發路徑務實並具備快速上架能力，系統依循「核心與策略、權重與社群、AI 智能輔助」三大階段漸進開發：
1. 核心功能與遊戲化策略定義（第一階段）：
首要任務是完成基礎健身紀錄的畫面呈現與核心計劃動作庫建構。採用 Flutter 框架開發，並透過 FlutterFire 插件 串接 Firebase 雲端儲存。此階段重點在於建立 500+ 項健身動作的結構化標準，並確立「八角框架」遊戲化策略。為應對網路不穩環境，前端導入本地緩存機制，確保訓練數據能即時紀錄並於連網後自動同步，維持流暢的使用體驗。
2. 權重運算與遊戲化社群機制（第二階段）：
隨後導入「動態權重運算引擎」。該引擎部署於 Cloud Functions 中，負責根據動作難度、個人進步幅度及安全性指標（如偵測過度訓練），將原始訓練數據轉化為具備情感價值的數位資產。此階段同步完成遊戲化視覺（如雷達圖、獎盃架）與社交系統。除了核心的師徒機制外，也整合了FCM (Firebase Cloud Messaging)推播機制與動態社群互動，強化社群黏性。
3. AI 智能教練與安全性強化（第三階段）：
最後透過 Google AI Studio 整合 Gemini API 實作生成式 AI 功能。為了消除 AI 幻覺並確保專業性，系統採用 RAG (檢索增強生成) 接地技術：Cloud Functions 會動態檢索 Firestore 中的專業動作資料庫進行校準，強制將 AI 建議限制在專業範疇內。此架構能實現具備「艾佛烈效應」的極致個人化體驗，並隨時透過雲端更新動作知識。最後部署 Google Cloud Secret Manager 與 Firebase App Check，透過設備合法性驗證與 API 金鑰防護，確保系統達成商用級安全性。

### 3-2 系統軟、硬體需求與平台技術

本系統旨在建立高可靠性且具備 AI 運算能力的健身環境，針對開發端與終端使用者之需求，其技術規格與環境配置如下：
1. 系統軟體需求 (Software Requirements)
系統之開發與運行環境架構於現代化雲端工具鏈之上，確保開發效率與後續維護之穩定性：
* 開發環境：採用 Visual Studio Code (VS Code) 作為主要整合開發環境。
* 版本控制：依循 Git 標準流程，並使用 GitHub 進行程式碼版本控管。
* 開發框架：採用 Flutter 跨平台框架，僅需維護單一程式碼即可同時產出 iOS 與 Android 版本，極大化開發效益。
* AI 推理引擎：核心模型採用 Gemini 系列模型，透過 API 介面串接獲取生成式推理能力。
* 後端 SDK 整合：透過 FlutterFire 插件深度串接 Firebase 各項服務元件（含身分驗證、雲端資料庫及 App Check 安全防護）。
2. 系統硬體需求 (Hardware Requirements)
為確保遊戲化視覺效果（如雷達圖渲染）與 AI 智能報告之運算流暢度，建議硬體規格如下：
* 裝置類型：支援 iOS 或 Android 系統之行動裝置，並需具備生物辨識（FaceID / TouchID）功能以進行身分安全驗證。
* 作業系統：建議 iOS 15.0 或 Android 10.0 (API Level 29) 以上版本。
* 網路需求：具備穩定之 4G/5G 行動網路或 Wi-Fi 連線環境，以利 AI 智能計畫之即時校準與同步。
* 儲存空間：裝置需預留至少 200MB 以上空間，用於緩存動作資料庫、使用者健身資產與離線紀錄數據。

3. 平台技術說明 (Platform Technologies)
本系統結合了多項雲端運算技術，將產品建構為高性能之 SaaS (Software as a Service) 解決方案：
* BaaS 雲端服務模型：利用 Firebase 作為後端即時服務平台 (Backend as a Service)，整合身分驗證、雲端資料庫與即時同步功能，大幅降低基礎設施之維護成本。
* Serverless 無伺服器架構：透過 Cloud Functions 建立雲端邏輯運算中樞，處理核心權重邏輯與 AI 調度。這能確保計算結果的公正性與安全性，且系統可隨流量自動擴展，達成商用級運算效率。
* 應用程式安全防護：導入 Firebase App Check 技術與 Secret Manager，確保僅有經授權的合法客戶端可調用 API 資源，杜絕非法流量與金鑰外洩風險。
* 數據接地技術 (Grounding)：透過技術手段將 LLM 生成邏輯與 Firestore 中的 500+ 項結構化動作資料庫進行動態校準，確保 AI 建議內容之精準度與物理安全性。
* 本地緩存技術：前端實作數據本地化存取邏輯，確保在健身房弱網環境下仍能進行流暢紀錄，顯著提升使用者的操作體驗與系統穩定度。
